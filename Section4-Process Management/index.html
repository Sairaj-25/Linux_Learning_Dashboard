⚙️ What Is a Process?

A process is an instance of a running program. When you execute a command, the Linux kernel creates a process, allocates memory, assigns a unique PID (Process ID), and schedules it for CPU time.

Key Process Attributes

AttributeDescription
PIDProcess ID – unique integer assigned by the kernel
PPIDParent PID – the process that spawned this one
UID / EUIDReal / Effective user ID owning the process
GID / EGIDReal / Effective group ID
TTYControlling terminal (? = none / daemon)
CPU %Current CPU utilisation
MEM %Physical RAM share
Priority / NiceScheduling priority; nice value alters it
CMDThe command name / path

Types of Processes

TypeDescription
ForegroundAttached to the terminal; blocks the shell until it exits
BackgroundLaunched with &; terminal stays usable
DaemonBackground service with no controlling terminal (TTY = ?)
ZombieExited but parent has not yet read its exit status
OrphanParent died first; adopted by init (PID 1)
💡 PID 1 is always systemd (or init on older systems). Every other process is a descendant — Linux uses a tree structure.

🔍 Viewing Processes – ps

ps (process snapshot) is the go-to command for listing processes at a point in time. It supports two flag styles: Unix (-) and BSD (no dash).

Common ps Invocations

# BSD-style: all processes, user-friendly columns
ps aux

# Unix-style: every process, full format
ps -ef

# Show a full process tree (forest)
ps -ef --forest

# Show specific columns only
ps -eo pid,ppid,user,stat,nice,pcpu,pmem,cmd

# Filter by process name
ps -C nginx

# Show threads
ps -eLf

# Sort by CPU usage (descending)
ps aux --sort=-%cpu | head -10

ps aux Column Reference

ColumnMeaning
USEROwner of the process
PIDProcess ID
%CPUCPU usage since process start
%MEM% of physical RAM used
VSZVirtual memory size (KB)
RSSResident set size – physical RAM used (KB)
TTYControlling terminal
STATProcess state (see above)
STARTTime/date process started
TIMECumulative CPU time consumed
COMMANDFull command with arguments

Finding Processes – pgrep / pidof

# Print PIDs matching a name pattern
pgrep nginx

# Print PID + name (verbose)
pgrep -l ssh

# Match exact name
pgrep -x bash

# Match by user
pgrep -u sairaj

# pidof – exact binary name
pidof sshd

📊 top & htop – Real-Time Monitoring

top

top gives a live, auto-refreshing view of system resource usage. It updates every 3 s by default. 

top                    # launch top
top -u sairaj         # show only processes of a specific user
top -p 1234,5678     # watch specific PIDs
top -b -n 1           # batch mode – single snapshot (good for scripts)

top Interactive Keys

KeyAction
qQuit
kKill a process (prompts for PID + signal)
rRenice a process
PSort by CPU %
MSort by Memory %
TSort by cumulative Time
1Toggle per-CPU stats
uFilter by user
HToggle threads view
fOpen field/column selector

zToggle color

🧵 Jobs & Background Execution

The shell tracks jobs – processes it has started. You can suspend, background, foreground, or disown them.

Running Jobs

# Start a command in the background
sleep 300 &

# List current shell's jobs
jobs
jobs -l    # include PID

# Bring job 1 to the foreground
fg %1

# Send job 1 to the background (if currently stopped)
bg %1

# Suspend the current foreground job
Ctrl+Z

# Detach job so it survives terminal close
disown %1

nohup – Immunity to HUP Signal

# Run command that keeps going after you log out
nohup ./long_script.sh &

# Output goes to nohup.out by default; redirect it:
nohup ./long_script.sh > /tmp/script.log 2>&1 &

screen / tmux (session managers)

### screen ###
screen                        # start new session
Ctrl+a d                     # detach
screen -r                    # reattach
screen -ls                   # list sessions

### tmux ###
tmux                         # start new session
Ctrl+b d                     # detach
tmux attach                  # reattach
tmux ls                      # list sessions
✅ Prefer tmux for persistent sessions on remote servers. Use nohup for one-off long-running commands.

📡 Process Signals

Signals are software interrupts sent to processes. The kernel, other processes (with the right permissions), or the user can send them. 

# List all signals
kill -l

Most Important Signals

NumberNameDefault ActionNotes
1SIGHUPTerminateHangup – often used to reload config
2SIGINTTerminateKeyboard interrupt (Ctrl+C)
3SIGQUITCore dumpCtrl+\
9SIGKILLTerminate (force)Cannot be caught or ignored
15SIGTERMTerminateDefault kill signal; allows graceful shutdown
17SIGCHLDIgnoreChild stopped or exited
18SIGCONTContinueResume a stopped process
19SIGSTOPStopCannot be caught or ignored
20SIGTSTPStopKeyboard stop (Ctrl+Z)
10SIGUSR1TerminateUser-defined signal 1
12SIGUSR2TerminateUser-defined signal 2
⚠️ SIGKILL (9) immediately destroys the process without giving it a chance to clean up. Always try SIGTERM (15) first, then SIGKILL as a last resort.

🔫 kill / killall / pkill

kill – Send Signal by PID

# Send SIGTERM (15) to PID 1234
kill 1234

# Send SIGKILL to PID 1234
kill -9 1234
kill -SIGKILL 1234     # same thing

# Reload config of a daemon (nginx, sshd, etc.)
kill -HUP $(cat /var/run/nginx.pid)

# Kill all processes in a group (negative PID)
kill -TERM -5678

killall – Kill by Process Name

# Kill all processes named "firefox"
killall firefox

# Force kill
killall -9 firefox

# Kill only processes owned by a specific user
killall -u sairaj python3

# Verbose – report what was killed
killall -v nginx

# Interactive – confirm before killing
killall -i nginx

pkill – Pattern-Based Kill

# Kill by name pattern (regex supported)
pkill nginx

# Force kill matching pattern
pkill -9 nginx

# Kill by user
pkill -u sairaj

# Kill exact name match only
pkill -x bash

# Send SIGHUP to reload
pkill -HUP sshd

Comparison: kill vs killall vs pkill

CommandTargetBest For
killPIDPrecise; use when you know the exact PID
killallExact process nameKilling all instances of a named binary
pkillName pattern / attributesFlexible matching (user, terminal, regex)

⚖️ Process Priority – nice & renice

Linux uses two related values to control scheduling priority:

ValueRangeNotes
Nice value-20 to +19Lower = higher priority. Set at launch or changed with renice.
Priority (PRI)0–139Kernel's real scheduling priority. PRI = 20 + nice for normal processes.
💡 Only root can set a negative nice value (increase priority). Regular users can only lower priority (raise the nice value).

nice – Launch with a Priority

# Launch with niceness +10 (lower priority than default 0)
nice -n 10 ./backup.sh

# Launch with highest possible priority (root only)
sudo nice -n -20 ./critical.sh

# Default nice value when no flag given is +10
nice ./script.sh

renice – Change Priority of a Running Process

# Set nice value to +5 for PID 1234
renice +5 -p 1234

# Set nice value to -5 (root required)
sudo renice -5 -p 1234

# Renice all processes owned by a user
renice +10 -u sairaj

# Renice an entire process group
renice +5 -g 1234

# Verify the change
ps -o pid,ni,pri,cmd -p 1234

CPU Affinity – taskset

# Launch a command pinned to CPU 0 and 1
taskset -c 0,1 ./program

# Pin an existing process to CPU 2
taskset -cp 2 1234

# Check current affinity
taskset -cp 1234

📂 /proc Filesystem

/proc is a virtual filesystem maintained by the kernel. Every running process has a directory /proc/<PID>/ containing live information.

Useful /proc/<PID> Entries

File / DirContains
/proc/<PID>/statusHuman-readable process info (name, PID, PPID, state, UID, memory)
/proc/<PID>/cmdlineFull command that started the process (NUL-separated)
/proc/<PID>/environEnvironment variables at process start
/proc/<PID>/mapsMemory maps (virtual address → file/lib)
/proc/<PID>/fd/Symlinks to open file descriptors
/proc/<PID>/net/Network info for the process's namespace
/proc/<PID>/ioI/O statistics (reads/writes)
/proc/<PID>/limitsResource limits (ulimits)
/proc/<PID>/threads/Per-thread directories

System-wide /proc Files

# Current logged-in users + uptime
cat /proc/uptime

# CPU info
cat /proc/cpuinfo

# Memory info
cat /proc/meminfo

# Kernel command line at boot
cat /proc/cmdline

# Loaded kernel modules
cat /proc/modules

# Read a specific process's command
cat /proc/1/cmdline | tr '\0' ' '

🛠️ Process Monitoring Tools

vmstat – Virtual Memory Statistics

# Refresh every 2 seconds, 5 times
vmstat 2 5

# Display timestamps
vmstat -t 1

iostat – CPU and I/O Stats

iostat                  # one snapshot
iostat -x 2            # extended stats, refresh every 2 s
iostat -d sda 2        # only disk sda

lsof – List Open Files / Sockets

# All open files
lsof

# Open files by a specific process
lsof -p 1234

# Who has a specific file open?
lsof /var/log/syslog

# Network connections
lsof -i
lsof -i :80             # who's listening on port 80
lsof -iTCP -sTCP:LISTEN

strace – System Call Tracer

# Trace syscalls of a new command
strace ls

# Attach to a running process
strace -p 1234

# Summary of syscall counts
strace -c ./program

# Only trace specific syscalls
strace -e trace=open,read,write ./program

Additional Useful Commands

CommandPurpose

watch -n 1 'ps aux'Run ps every second (poor-man's top)

pidstat 2Per-process CPU/IO stats (sysstat)
