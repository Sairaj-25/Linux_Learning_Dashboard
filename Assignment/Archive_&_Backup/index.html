PART I: STEP-BY-STEP PROCEDURE - BACKUP LIFECYCLE

STEP 1: Identify Data & Strategy

Determine what needs to be backed up (e.g., /home, /etc) and choose a compression algorithm based on CPU vs. space trade-offs.

Rationale: Compressing text configs yields massive savings, while compressing already-compressed video files wastes CPU. Understanding your data dictates your tar flags.

STEP 2: Archive Creation

Use tar with create (-c), verbose (-v), file (-f), and compression (-z or -j) options.

Rationale: Combining files into a single stream makes writing to tape or network faster, and compression reduces storage footprints. The -f flag must always be last before the filename.

STEP 3: Verification

Always test the archive without extracting using tar -t (list).

Rationale: A backup that cannot be restored is useless. Verifying the table of contents ensures the header and structure are intact before moving it to cold storage.

STEP 4: Restoration

When disaster strikes, use tar -x to extract. Be extremely careful about your current working directory.

Rationale: Extracting an archive of /etc into your home directory creates ~/etc, which is harmless. Extracting it from / overwrites system configs. Path awareness is critical.

PART II: COMMANDS AND EXPLANATIONS

Core Functional Modes (Must pick one)

  • -c: Create a new archive.
  • -x: eXtract files from an archive.
  • -t: List the contents of an archive.
  • -r: Append files to the end of an uncompressed archive.

Essential Modifiers

  • -f: File. Specifies the archive filename. Must be last before the name (e.g., -cvf backup.tar, not -cfv).
  • -v: Verbose. Prints the filenames being processed to stdout.

Compression Algorithms

  • -z: gzip. Fastest, standard compression. Extension: .tar.gz or .tgz.
  • -j: bzip2. Slower but better compression than gzip. Extension: .tar.bz2.
  • -J: xz. Slowest but highest compression ratio. Extension: .tar.xz.

Syntax Examples

# Create gzip archive
tar -czvf archive.tar.gz /path/to/folder

# Extract gzip archive
tar -xzvf archive.tar.gz

# List contents
tar -tzvf archive.tar.gz

PART III: ALTERNATIVE COMMANDS & METHODS

1. tar vs zip
  • tar: A tape archiver. It combines files into a single uncompressed stream, then applies compression to the whole stream. Preserves Linux permissions, ownership, and symlinks perfectly.
  • zip: Compresses files individually before archiving. Good for cross-platform sharing (Windows/Mac), but loses Linux permissions and symlink data unless special flags are used.
2. gzip vs bzip2 vs xz Trade-offs
AlgorithmFlagSpeedCompression RatioUse Case
gzip-zFastestLowestDaily backups, network transfer
bzip2-jMediumMediumLong-term storage where CPU time isn't critical
xz-JSlowestHighestSoftware distribution (kernel sources), archiving
3. cpio

An older alternative to tar commonly used in RPM packages and initramfs. It reads paths from standard input (find /path | cpio -o), making it more pipeline-friendly than tar, but its syntax is less intuitive for general use.

Section B: Short Answer Questions

1. Define archiving.

Archiving is the process of combining multiple files and directories into a single file. It does not inherently reduce file size; it simply groups them together for easier portability or storage.

2. Define backup.

A backup is a copy of important data stored separately from the original, used to restore the original in the event of data loss, corruption, or system failure.

3. Purpose of the tar command?

To create, maintain, modify, and extract files from archives. It is the standard utility for grouping files in Linux while preserving file permissions, ownership, and directory structures.

4. Basic syntax of tar.
tar [options] [archive-name] [files-or-directories]
5. Explain the -f option.

-f stands for "File". It specifies the filename of the archive being created, listed, or extracted. It must be followed immediately by the archive name.

6. Difference between archiving and compression?

Archiving combines many files into one without shrinking them. Compression uses algorithms to reduce the file size by eliminating redundant data. tar archives; gzip/bzip2 compress.

7. How do you extract a compressed archive?

You use the -x (extract) flag along with the specific compression flag (e.g., -z for gzip, -j for bzip2). Example: tar -xzvf archive.tar.gz.

8. Purpose of -tvf?

-t lists the contents, -v shows details (permissions, size, owner), and -f specifies the file. Together, they display a detailed listing of an archive's contents without extracting them.

9. Why are backups important?

Hardware fails, humans make mistakes, and security breaches happen. Backups ensure business continuity, prevent permanent data loss, and allow system rollbacks after failed updates.

10. Two distros where tar is available?

Ubuntu and Red Hat Enterprise Linux (RHEL). (It is universal across virtually all Linux/Unix systems).

Section C: Command Writing

1. Create backup.tar for myfolder.
tar -cvf backup.tar myfolder/
2. Create gzip archive backup.tar.gz.
tar -czvf backup.tar.gz myfolder/
3. Extract backup.tar.
tar -xvf backup.tar
4. Extract backup.tar.gz.
tar -xzvf backup.tar.gz
5. View contents of backup.tar.
tar -tvf backup.tar
6. Backup multiple files (file1, file2, file3).
tar -cvf backup.tar file1 file2 file3
7. Backup /etc directory.
# Typically requires root
sudo tar -czvf etc_backup.tar.gz /etc
8. Restore backup to root directory.
# Change to root, then extract
cd /
sudo tar -xzvf /path/to/backup.tar.gz

Danger Extracting to / overwrites existing system files.

9. Daily backup of /home with date.
tar -czvf /backup/home_$(date +%F).tar.gz /home
10. Command using bzip2 compression.
tar -cjvf archive.tar.bz2 myfolder/

Section E: Scenario-Based Questions

1. Combine project files into single file for transfer.

Use tar to create an uncompressed archive (fastest) or a compressed one (smaller size):

tar -cvf project.tar /path/to/project/
# OR for smaller size:
tar -czvf project.tar.gz /path/to/project/
2. Daily compressed backup of /home.

Combine tar with the date command for unique daily filenames:

tar -czvf /backup/home_backup_$(date +%Y-%m-%d).tar.gz /home
3. Extract data.tar.gz received from manager.
tar -xzvf data.tar.gz

The files will extract into the current directory.

4. Backup /etc before system maintenance.
sudo tar -czvf /root/etc_maintenance_backup.tar.gz /etc

Store it in /root or an external drive so it isn't affected by the maintenance.

5. Check contents without extracting.
tar -tzvf archive.tar.gz

PART V: INTERVIEW QUESTIONS AND ANSWERS

Q1: What is the difference between tar and zip?

tar is an archiver; it bundles files into one stream. Compression is an optional add-on (-z, -j). It flawlessly preserves Linux file permissions, UID/GID, and symlinks.

zip is a compressor and archiver. It compresses files first, then archives them. It is cross-platform (Windows/Mac) but traditionally loses Linux permission data, making it dangerous for system backups.

Q2: Why must the -f flag always be the last option in a tar command?

Because the argument immediately following -f is interpreted as the name of the archive file. If you type tar -cfv archive.tar, tar will try to create an archive named v and will try to add files named archive.tar (and fail). The correct syntax is tar -cvf archive.tar.

Q3: How do you extract a single specific file from a massive tar.gz archive?

Append the path of the file you want at the end of the extract command. You must know the exact path inside the archive (use -t to find it).

# Find the path
tar -tzvf backup.tar.gz | grep myfile.txt
# Extract just that file
tar -xzvf backup.tar.gz path/to/myfile.txt
Q4: How do you preserve SELinux contexts and ACLs when creating a tar archive?

Standard tar does not preserve SELinux contexts or Access Control Lists (ACLs) by default. You must use the --selinux and --acls flags. For a full backup, use:

tar --selinux --acls --xattrs -czvf backup.tar.gz /data
Q5: What happens if you extract an archive that contains absolute paths (e.g., /etc/passwd)?

By default, modern versions of tar strip the leading slash (/) and extract to the current working directory (e.g., ./etc/passwd). This is a safety feature to prevent accidentally overwriting critical system files. However, older versions of tar would extract directly to the absolute path, potentially destroying the live system. Always check an archive's contents with -t before extracting.
