chmod, chown, chgrp & umask

PART I: STEP-BY-STEP PROCEDURE - PERMISSIONS LIFECYCLE

STEP 1: Assessment

Use ls -l to inspect current ownership and permissions.

Rationale: You cannot secure what you do not understand. Identifying the current owner, group, and mode is the prerequisite to any modification.

STEP 2: Ownership Configuration

Modify ownership using chown and chgrp before setting permissions.

Rationale: Permissions are tied to the owner and group. If you set strict permissions and then change the owner, the new owner gets the rights. Always define "who" before defining "what they can do".

STEP 3: Permission Assignment

Use chmod to enforce the principle of least privilege.

Rationale: Apply the minimum permissions required for the resource to function. Over-privileging (e.g., chmod 777) is a primary vector for security breaches and accidental data loss.

STEP 4: Umask Hardening

Review and configure umask in shell profiles to ensure future files inherit secure defaults.

Rationale: Manually securing existing files is useless if newly created files are left wide open. A strict umask prevents permissive defaults at the moment of creation.

PART II: COMMANDS AND EXPLANATIONS

chmod (Change Mode)

Syntax: chmod [options] mode file

Purpose: Alters the read, write, and execute permissions for the file owner, group, and others.

# Numeric (Octal)
chmod 755 script.sh

# Symbolic
chmod u+x script.sh
chmod o-w config.yaml

chown (Change Owner)

Syntax: chown [options] owner[:group] file

Purpose: Changes the user and/or group ownership of a file or directory. Requires root privileges.

# Change owner only
sudo chown alice file.txt

# Change owner and group simultaneously
sudo chown alice:developers file.txt

chgrp (Change Group)

Syntax: chgrp [options] group file

Purpose: Changes the group ownership. Can be run by the file owner (if they belong to the target group) without root.

chgrp developers project_alpha/

umask (User File Creation Mode Mask)

Syntax: umask [value]

Purpose: Sets the default permission mask for newly created files and directories. It subtracts permissions from the base system defaults (777 for dirs, 666 for files).

umask          # View current mask (e.g., 0022)
umask 027      # Set strict mask temporarily

PART III: ALTERNATIVE COMMANDS & METHODS

1. Numeric vs. Symbolic chmod
  • Numeric (755): Absolute. Overwrites all permission bits. Best for scripting exact desired states.
  • Symbolic (u+x): Relative. Modifies specific bits without affecting others. Best for incremental changes (e.g., adding execute without touching read/write).
2. chown vs. chgrp

chown user:group changes both at once and requires root. chgrp group changes only the group and can be run by a regular user if they own the file and are a member of the target group.

3. Standard Permissions vs. ACLs (setfacl)

Standard chmod limits you to one owner, one group, and others. If you need to give specific access to a second group or a specific user without changing ownership, Access Control Lists (setfacl) are required.

# Grant user 'bob' read access without changing ownership
setfacl -m u:bob:r file.txt

PART IV: ASSIGNMENT SOLUTIONS

Section A: Short Answer Questions

1. What are file permissions in Linux?

Rules that determine who can read, modify, or execute a file, and who has access to directory contents.

2. Purpose of file permissions?

To enforce security, data integrity, and privacy by restricting unauthorized access or modification of system and user data.

3. What is umask?

A 4-digit octal mask that controls the default permissions assigned to newly created files and directories.

4. Default umask values?

Root user: 022. Normal user: 002 (on modern systems with User Private Groups).

5. Default base permissions?

Files: 666 (rw-rw-rw-). Directories: 777 (rwxrwxrwx).

6. What does r, w, x represent?

r = Read, w = Write, x = Execute.

7. Octal values for r, w, x?

Read = 4, Write = 2, Execute = 1.

Section B: Conceptual Questions

1. How Linux calculates final permissions using umask.

Subtract the umask value from the base default permissions.
For files: 666 - umask.
For directories: 777 - umask.
Advanced Note: Mathematically, it's a bitwise AND with the inverse of umask. This prevents files from ever getting execute permissions by default, even if umask is 000 (666 & ~000 = 666).

2. Differentiate permissions for user, group, others.

User (Owner): The creator/owner of the file. Has the most specific rights.
Group: Users belonging to the file's assigned group. Used for collaborative access.
Others: Everyone else on the system. Should have the most restrictive rights.

3. Symbolic vs. Numeric methods.

Numeric: Uses octal numbers (0-7) representing binary permission bits (e.g., 755). Sets absolute permissions for all three categories simultaneously.
Symbolic: Uses letters (u,g,o,a) and operators (+,-,=) to modify specific bits (e.g., g+w). Allows incremental changes without knowing the current state.

4. What happens if execute permission is removed from a directory?

Users cannot cd into the directory (enter it), nor can they access any files inside it, even if they know the filenames and have read permissions on those files. Execute on a directory means "search/traverse" permission.

5. Why is umask important for security?

Without a restrictive umask (like 022 or 077), new files might be created as world-readable/writable, exposing sensitive data (like SSH keys or config files) to all local users.

1. Calculate: Default = 666, umask = 022

666 - 022 = 644 (rw-r--r--)

2. Calculate: Default = 777, umask = 002

777 - 002 = 775 (rwxrwxr-x)

3. Convert rwxr-xr-- to octal.

Owner (rwx) = 4+2+1 = 7
Group (r-x) = 4+0+1 = 5
Others (r--) = 4+0+0 = 4

Section E: Viva Questions

1. Difference between chmod and chown?

chmod changes what can be done with a file (permissions). chown changes who owns the file (user/group identity).

2. What does -R option do?

Recursive. It applies the command to the target directory and all files/subdirectories within it.

3. Meaning of chmod a=rwx?

Sets the permissions to read, write, and execute for all categories (User, Group, Others). It overwrites existing permissions.

4. Can a file have execute permission without read permission?

Yes (e.g., 111 or 311). However, it is usually useless for scripts (bash needs to read the script to execute it). It only works for compiled binaries.

5. Role of group in file permissions?

It enables collaborative access. Users in the same group can be granted specific rights (e.g., read/write) without exposing the file to "others" or granting individual user ownership.

PART V: INTERVIEW QUESTIONS AND ANSWERS

Q1: What is the difference between chown user:group and chown user.group?

Functionally, they are identical in modern Linux. Both change the owner and group simultaneously. However, the colon : is the POSIX standard and preferred syntax, as the dot . can be confused with usernames containing dots (e.g., first.last).

Q2: Explain the security risk of chmod 777 and how to fix it for a web directory.

777 allows anyone to write to the directory, meaning an attacker could replace your index.html with malware. For a web directory, 755 (owner write, others read/execute) is standard. For files, 644. Fix: find /var/www -type d -exec chmod 755 {} \; and find /var/www -type f -exec chmod 644 {} \;

Q3: What happens if a directory has --x (execute only) permissions?

A user can pass through the directory (cd into it) if they know the exact name of a file inside, they can read/execute it (if the file permissions allow). However, they cannot list the directory contents (ls will fail). This is often used for drop-box directories.

Q4: Why does the umask subtraction method fail for umask 003 and base 666?

Simple subtraction: 666 - 003 = 664 (rw-rw-r--).
Actual Linux result: 664.
Wait, that looks right? Let's try umask 033. Subtraction: 666 - 033 = 633. Actual result: 644.
Because the execute bit is never set by default on files, umask uses a bitwise AND. 666 & ~033 = 644. Simple subtraction fails when umask attempts to remove execute bits that aren't there in the base 666.

Q5: How do you permanently change the umask for a specific user?

Add the umask command to the user's shell profile files, such as ~/.bashrc or ~/.bash_profile. For system-wide changes, modify /etc/profile or /etc/login.defs.
