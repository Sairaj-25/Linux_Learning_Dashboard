PART I: STEP-BY-STEP PROCEDURE - FOR LOOP LIFECYCLE

STEP 1: List Evaluation

Bash evaluates the list of items provided after the in keyword (e.g., 1 2 3 4 5 or *.txt).

Rationale: The loop must first determine what it is going to iterate over. If no list is provided, it defaults to the positional parameters ($@).

STEP 2: Variable Assignment

The first item in the list is assigned to the loop variable (e.g., i).

Rationale: This creates the reference the script will use inside the loop body to manipulate the current item.

STEP 3: Execution Block (do...done)

The commands between do and done are executed using the current value of the loop variable.

Rationale: This is the core logic of the loop that processes the item.

STEP 4: Iteration & Termination

Once the block finishes, Bash moves to the next item in the list, assigns it to the variable, and repeats. If no items are left, the loop exits.

Rationale: Ensures the loop processes every item exactly once and then stops, preventing infinite loops (unless explicitly designed).

PART II: COMMANDS & SYNTAX EXPLANATIONS

Standard For Loop Syntax

for variable in item1 item2 item3
do
  # Commands to execute for each item
done

Purpose: Iterates over a discrete list of strings, numbers, or file expansions.

Range-Based For Loop (Brace Expansion)

for i in {1..10}
do
  echo $i
done

Purpose: Generates a sequence of numbers or characters. Tip You can add an increment: {1..10..2} (1 to 10, step 2).

C-Style For Loop

for (( i=1; i<=5; i++ ))
do
  echo $i
done

Purpose: Ideal for complex mathematical iterations where you need precise control over initialization, condition, and increment steps.

Arithmetic Expansion $((...))

Syntax: result=$(( variable + 1 ))

Purpose: Used inside loops to perform integer math (like summing numbers or incrementing counters). Bash does not support floating-point math natively.

PART III: ALTERNATIVE COMMANDS & METHODS

1. for vs while loops
  • for: Best when you know the exact list of items or the number of iterations beforehand (e.g., iterating over files, a sequence of numbers).
  • while: Best when the number of iterations is unknown and depends on a dynamic condition (e.g., reading a file line by line until EOF, waiting for a service to start).
2. {1..10} vs seq 1 10
  • {1..10}: Built-in Bash brace expansion. Faster because it doesn't spawn a subshell. Limitation: Cannot use variables inside braces (e.g., {1..$n} will NOT work).
  • seq 1 10: External command. Slightly slower. Advantage: Works perfectly with variables (e.g., seq 1 $n).
3. for loop vs xargs

For processing large numbers of files or lines, piping to xargs is often much faster than a for loop. A for loop spawns a new subshell for *every iteration* if executing an external command, while xargs bundles arguments to execute the command fewer times.

PART IV: ASSIGNMENT SOLUTIONS

Section A: Theory Questions

1. What is a for loop in Bash?

A for loop is a control flow statement that iterates over a list of items, executing a block of code for each item in the list.

2. Why do we use a for loop?

To automate repetitive tasks, process multiple files, iterate through lists of variables, or execute a block of code a specific number of times without writing duplicate code.

3. Basic syntax of a for loop.
for variable in list
do
  commands
done
4. Purpose of do and done?

do marks the beginning of the loop body (the commands to be executed). done marks the end of the loop block and tells Bash to go back to the top of the loop for the next item.

5. How does a for loop help in automation?

It allows sysadmins to perform batch operations—like renaming 100 files, checking the status of 50 servers, or processing thousands of lines of logs—with just a few lines of code instead of manually executing commands one by one.

#!/bin/bash 
for i in 1 2 3 4 5 
do 
  echo $i 
done 
# Note: The list is space-separated integers.
2. Print Student Names
#!/bin/bash 
for name in Imran Abdul Rahim Ali 
do 
  echo "Student: $name" 
done 
# Note: Strings don't need quotes here if they have no spaces.
3. Print Even Numbers
#!/bin/bash 
for i in 2 4 6 8 10 
do 
  echo $i 
done
4. Print Numbers Using Range
#!/bin/bash 
for i in {1..10} 
do
  echo $i 
done 
# Note: Brace expansion {start..end} is the standard Bash way to generate sequences.
5. Display Files in Current Directory
#!/bin/bash 
for file in * 
do 
  echo $file 
done 
# Note: The glob '*' expands to all files/dirs in the current directory.
6. Print Table of 5
#!/bin/bash 
for i in {1..10} 
do 
  echo "5 x $i = $((5*i))" 
done 
# Note: $((5*i)) performs integer arithmetic.
7. Sum of Numbers from 1 to 5
#!/bin/bash 
sum=0 
for i in {1..5} 
do 
  sum=$((sum+i)) 
done 
echo "Sum = $sum" 
# Note: The variable 'sum' is updated in each iteration.
8. Check Multiple Users
#!/bin/bash 
for user in root imran test 
do 
  id $user 
done
# Note: 'id' command returns user info if they exist, or error if they don't.

PART V: INTERVIEW QUESTIONS AND ANSWERS

Q1: Why does for i in {1..$n} fail, and how do you fix it?

Reason: Brace expansion happens *before* variable expansion in Bash. Therefore, Bash sees the literal string {1..$n} and doesn't expand it.

Fix: Use the C-style for loop or seq:

# Method 1: C-style loop
for (( i=1; i<=$n; i++ )); do ...

# Method 2: seq
for i in $(seq 1 $n); do ...

Q2: How do you handle filenames with spaces in a for loop?

If you do for file in *, filenames with spaces will be split into multiple iterations (e.g., "my file.txt" becomes "my" and "file.txt").

Fix: Always quote the variable inside the loop body, and for complex cases, use find with while read or adjust IFS:

# Better approach for spaces
find . -type f -print0 | while IFS= read -r -d $'\0' file; do
  echo "$file"
done
Q3: What is the difference between break and continue in a loop?

break: Immediately exits the loop entirely, skipping the rest of the iterations, and moves to the code after done.
continue: Skips the remaining commands in the current iteration and jumps straight to the next iteration of the loop.

Q4: How do you run a for loop in the background or in parallel?

Add an ampersand & after the command inside the loop to spawn a subshell. However, you should use wait after the loop to ensure the script doesn't exit before all background jobs finish.

for i in {1..10}; do
  heavy_command "$i" &
done
wait
echo "All parallel jobs completed!"
Q5: How do you create an infinite for loop in Bash?
# Method 1: C-style
for ((;;)); do
  echo "Running forever..."
  sleep 1
done

# Method 2: While loop (more common)
while true; do
  echo "Running forever..."
  sleep 1
done
